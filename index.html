Git

Definition:
Git is a distributed version control system that allows multiple people to work on a project simultaneously. It helps track changes in source code, facilitates collaboration, and provides a history of modifications.Git is like a time-traveling tool for code. It helps software developers keep track of changes they make to their code over time. If something goes wrong, you can easily go back in time to fix it.

Real-world Example:
Imagine you're working on a group project with friends. Instead of passing around copies of the entire project folder, you each have your own copy. Git allows you to make changes independently on your copy (branch), and when ready, you can merge your changes back into the main project (branch).
Imagine you're writing a story on your computer. Before making big changes, you take a snapshot of the current version. If you mess up, you can always go back to the snapshot and start again. Git does this for code.

GitHub

Definition:
GitHub is a web-based platform that uses Git for version control. It provides a place to host Git repositories, making it easy for multiple people to collaborate on projects. GitHub also offers features like issue tracking, pull requests, and more.GitHub is like a shared online space for your code. It's like having a virtual office where multiple people can collaborate on a project. GitHub makes it easy for everyone to see the latest changes, work on their own parts, and bring everything together.

Real-world Example:
Think of GitHub as a virtual office where your project is stored. Each team member has their own desk (branch) to work on specific tasks. When someone completes their work, they submit a "pull request" to bring their changes back to the main project. GitHub acts as the project manager, keeping everything organized.
Think of GitHub as a virtual office where everyone has their desk. Each person works on their part of a project without getting in each other's way. When they finish, they share their work with the team. GitHub keeps everything organized, like a central hub for teamwork.

Some Comman Git Commands

Command Description
git init Initializes a new Git repository. Creates a hidden subfolder for version control within your project.
git clone [repository_url] Creates a copy of a remote repository on your local machine.
git add [file] Adds changes in the working directory to the staging area, preparing them for the next commit.
git commit -m "[commit_message]" Records changes to the repository along with a short commit message describing the changes.
git status Shows the status of changes as untracked, modified, or staged.
git log Displays a log of all commits made in the repository, including commit hash, author, date, and commit message.
git branch Lists all local branches in the repository. The current branch is highlighted with an asterisk.
git branch [branch_name] Creates a new branch with the specified name. Allows you to work on different features or fixes without affecting the main codebase.
git checkout [branch_name] Switches to the specified branch. Enables you to work on different branches and features.
git branch --delete [branch_name] Delete the respective branch.
git merge [branch_name] Combines changes from the specified branch into the current branch. Integrates changes and resolves conflicts.
git remote -v Lists all remote repositories connected to your local repository, showing both fetch and push URLs.
git pull Fetches changes from a remote repository and merges them into the current branch.
git push [remote] [branch] Pushes changes from your local branch to the specified remote repository. Shares your changes with others.
git diff Shows differences between the working directory and the staging area, highlighting changes not yet staged.
git rm [file] Removes a file from both the working directory and the staging area. Stages the file deletion.
git reset [file] Unstages the specified file but leaves the changes in your working directory.
git log --graph --oneline --all --decorate Displays a condensed, graphical representation of the commit history. Useful for visualizing branches and merges.
git stash Temporarily saves changes that are not ready to be committed. Allows switching branches without committing incomplete work.
git fetch [remote] Fetches changes from a remote repository without merging them into the current branch. Use git merge or git pull to integrate changes.
git revert [commit] Creates a new commit that undoes the changes made in a previous commit. A safer alternative to git reset.
git reset --hard HEAD^ This command performs a "hard reset" to the commit before the last commit (HEAD^). The --hard option discards changes in your working directory and staging area, effectively removing the last commit.It discards changes without any way to recover them. .
git reset --soft HEAD^ This performs a "soft reset," preserving changes in your working directory and staging area. .

SSH Key Generation

To establish secure communication between your local machine and GitHub, follow these steps:

# Generate a new SSH key using the Ed25519 algorithm ssh-keygen -t ed25519 -C "your_email@example.com"

This command creates a new SSH key using the Ed25519 algorithm, enhancing security.

# Copy the generated public key to the clipboard clip < ~/.ssh/id_ed25519.pub

This command copies the generated public key to the clipboard, ready for pasting into GitHub settings.

Next, go to GitHub settings under SSH, click "Add SSH Key," and paste the generated keys. Finally, check the connection with:

# Check if the connection was successful ssh -T git@github.com

This command verifies if the SSH connection to GitHub is successful, ensuring secure authentication displaying Hi USERNAME! You've successfully authenticated, but GitHub does not > provide shell access.

Alernatively we can update the remote URL to use SSH using git remote set-url origin git@github.com:username/reponame.git

We can set username and email in local machine through cmd using git config --global user.name and git config --global user.email

Step Description Command
1. Create a Remote Repository: Follow platform-specific instructions to create a remote repository on GitHub.
2. Navigate to Your Local Repository: Open a terminal or command prompt and navigate to your local Git repository.use git init to initialize it as a GIT repo.
3. Connect the Remote Repository: git remote add origin [remote_repository_url]
4. Verify the Connection: git remote -v
5. Pull the Latest Changes: git pull origin main
6. Push Your Changes: git push -u origin main
